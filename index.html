react-rails

Why

The existing rails starter for Ruby on Rails are too huge for small to mid size applications. Also it still uses tradtional tooling from ages. But the react ecosystem and tooling support are vastly changed. Also the boilerplate code is still huge. SSR config is bit hard.

Benefits
  1. Small size (just rails + react with intertia).
  2. Just clone it, and get started.
  3. SSR support. (so SEO)
view

Dynamo helper

Why

This is already existing library to interact with DynamoDB, made few enhancements to performance and data loading capabilities

Benefits
  1. Paginate (cursor) over DynamoDB results (query only), instead of waiting for all documents to be load and return. The cursor-based pagination adds a level of precision to data querying, optimizing the performance of applications relying on the DynamoDB database.
  2. No over memory consumption and memory constrains for serverless enironments. Minimize the data size retrieved from the DynamoDB database by executing queries on specific partition
view

Thumbnail

Why

There are several libraries available to create thumbnails for videos. But they are huge in size. We only need limited functionality along with custom watermarking solution.

Benefits
